Process Properties

You can view additional details for a process by selecting it and using the Process->Properties menu item or the context menu. Any dynamic data, such as performance information, updates at the refresh rate currently selected for PROWiSe Manager. You can manually refresh dynamic information by typing F5 in a page.
  • Process identifier (ID)

    • A numerical identifier that uniquely distinguishes a process while it runs. ID Process numbers are reused, so they only identify a process for the lifetime of that process.
    * Process - an executable program, such as Windows Explorer.
  • Parent process ID (Ð.ID)

    • Process identifier of the process created this one.

  • CPU Time

    • The total processor time, in seconds, used by a process since it started.

  • CPU Usage

    • The percentage of time that a process used the CPU since the last update.

  • User Name

    • The name of the user whose session owns the process.

  • Base Priority

    • A precedence ranking that determines the order in which the threads of a process are scheduled for the processor. Use PROWiSe Manager to view and change base priorities.

  • Handle Count

    • The total number of handles currently open by this process. This number is the sum of the handles currently open by each thread in this process.
    * Handle - a value used to uniquely identify a resource, such as a file or registry key, so that a program can access it.
  • Thread Count

    • The number of threads running in a process.
    * Thread - an object within a process that runs program instructions. Threads allow concurrent operations within a process and enable one process to run different parts of its program on different processors simultaneously.
  • Session ID

    • The session ID that owns the process.

  • Commit Charge

    • Memory allocated to process.

  • Working Set

    • The current number of bytes in the working set of this process. The working set of a program is a collection of those pages in its virtual address space that have been recently referenced. It includes both shared and private data. The shared data includes pages that contain all instructions the application executes, including those in DLLs. As the working set size increases, memory demand increases.

  • Private Bytes

    • The current number of bytes this process has allocated that cannot be shared with other processes.

  • VM Size

    • The current size of the virtual address space the process is using. Use of virtual address space does not necessarily imply corresponding use of either disk or main memory pages.

  • Context Switch

    • The system scheduler maintains a queue of executable threads for each priority level. When a processor becomes available, the system performs a context switch. The steps in a context switch are:
    * Context - a structure that contains processor-specific register data. Windows uses it to perform various internal operations.
     1. Save the context of the thread that just finished executing.
    2. Place the thread that just finished executing at the end of the queue for its priority.
    3. Find the highest priority queue that contains ready threads.
    4. Remove the thread at the head of the queue, load its context, and execute it.

    The most common reasons for a context switch are:
    1. The time slice has elapsed.
    2. A thread with a higher priority has become ready to run.
    3. A running thread needs to wait.

  • CSw Delta

    • Context Switch Delta since the last update.



  • Time Stamp

    • The time stamp for an image indicates the date and time that the image was created by the linker. The value is represented in Universal Coordinated Time.
