The Services view

Types of services:
  Applications
     Own process - Service that runs in its own process.
     Shared process - Service that shares a process with other services.

  Drivers
     File system driver.
     Device driver.

   Interactive - Application that can interact with the desktop.


You can see the following information about service:
  • Service Name

    • Used by Windows to identify the service.

  • Display Name

    • Used by user interface programs to identify the service.

  • Status

    • The current status of the service.

  • Startup

    • Specifies when to start the service. This member can be one of the following values:
      Boot - Specifies a device driver started by the operating system loader.
      System - Specifies a device driver started by the I/O system after boot devices such as the boot file system and disk driver have been initialized. The device driver started using this flag is not critical to the system boot.
      Auto - Specifies a device driver or service started by the service control manager automatically during system startup.
      Demand - Specifies a device driver or service started by the service control manager when a process calls the StartService function.
      Disabled - Specifies a device driver or service that can no longer be started.

  • Log On As

    • The account name in the form of "DomainName\Username", which the service process will be logged on as when it runs. Services of type 'Shared process' are not allowed to specify an account other than LocalSystem. If account isn't specified, the service will be logged on as the "LocalSystem" account.

  • Company Name

    • The company that produced the file.

  • Path to file

    • The path to the service binary file.
    If path is received but file doesn't exist, a service show in dark grey color.
    If path is not received, PROWiSe Manager looks for "%SYSTEM_DIR%\Drivers\SERVICE_NAME.sys". If file exist, this path with an exclamation character at the beginning will be used.

  • Load-Order group

    • The name of an load ordering group of which this service is a member. If the string is empty, the service does not belong to a group.

  • Tag in group

    • A unique tag value for this service in the group. Tag can be used for ordering service startup in a load ordering group by specifying a tag order vector in the registry located at: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GroupOrderList
    Tags are only evaluated for Drivers that have Boot or System start types.

  • Depends On

    • Names of services or load ordering groups that must start before this service. Group name begins with plus sign.

  • Description

    • Description of the service. The description is simply a comment that explains the purpose of the service. If the service has no description, then description of its file will be used.

  • Accept Shutdown

    • Specifies whether the system should notify the service when shutdown occurs.

  • Error Control

    • Specifies the severity of the error if this service fails to start during startup, and determines the action taken by the startup program if failure occurs.
      Ignore - The startup (boot) program logs the error but continues the startup operation.
      Normal - The startup program logs the error and displays a message but continues the startup operation.
      Severe - The startup program logs the error. If the last-known-good configuration is being started, the startup operation continues. Otherwise, the system is restarted with the last-known-good configuration.
      Critical - The startup program logs the error, if possible. If the last-known-good configuration is being started, the startup operation fails. Otherwise, the system is restarted with the last-known-good configuration.

Start, Stop, Pause, Resume
During startup a service specifies which of these messages it handles.

Services counter (in status bar)
Represents amount of services in the list in the following format: Total/Active/AutoStart. Next (in brackets) is the amount of services with invalid path to executable file.
* AutoStart is amount of services with one of the following startup-type: Auto, Boot, System.


Context Menu items:
  • Properties

    • This selection opens a property dialog that shows you more information about selected service.

  • File Properties

    • This selection opens a property dialog that shows you information about a file of the selected service.

  • Launch Depends

    • If PROWiSe Manager finds the Dependency Walker tool, it will be started with the binary file of selected service as a command-line argument.

  • Delete

    • This selection marks the specified service for deletion from the Service Control Manager database and deletes its file from the disk if possible.
