Performance Information

The Performance tab displays a dynamic overview of your computer's performance.

  • CPU Usage

    • A graph indicating the percentage of time the processor is working. This counter is a primary indicator of processor activity. View this graph to see how much processing time you are using. If this graph displays a high percentage, it means that one or more processes are use many cpu time. In this case your computer may running slowly.

  • CPU Usage History

    • A graph showing how busy the CPU has been over time. The sampling displayed in the graph depends on the value you select for the Update Speed on the View menu.

  • Memory Usage

    • Commit

      • A graph showing commit charge. When committed virtual memory, reaches the system Commit Limit, applications and the system become unstable. The Commit Limit is the sum of most of physical memory and the sizes of any paging files.
    • RAM Usage

      • A graph showing physical memory usage.

  • Totals for the number of handles, threads è processes, running on the computer.

    • * Handle - a value used to uniquely identify a resource, such as a file or registry key, so that a program can access it. * Thread - an object within a process that runs program instructions. Threads allow concurrent operations within a process and enable one process to run different parts of its program on different processors simultaneously. * Process - an executable program, such as Windows Explorer, or a service.
  • Physical Memory (Kb).

    • * Physical Memory (also called RAM) - memory installed on your computer. Information stored in RAM is lost when the computer is turned off. Available represents the amount of free memory that is available for use.
    The System Cache shows the current physical memory used to map pages of open files.

  • Kernel Memory (Kb).

    • * Kernel Memory - memory used by the operating system kernel and device drivers. Paged is memory that can be copied to the paging file, thereby freeing the physical memory. The physical memory can then be used by the operating system.
    Nonpaged is memory that remains resident in physical memory and will not be copied out to the paging file.

  • Commit Charge (Kb).

    • Memory allocated to programs and the operating system. Because of memory copied to the paging file, called virtual memory, the value listed under Peak may exceed the maximum physical memory.

  • Memory page size and allocation granularity (Kb).

    • Allocation granularity specifies the granularity with which virtual memory is allocated.

  • Paging information.

  • Context Switch Delta and I/O.
