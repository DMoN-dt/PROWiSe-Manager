Service Properties: Load And Logon

  • LogOn

    • Specifies an Account (User Name) under which a service will run.

    • LocalSystem
      This account is not recognized by the security subsystem. It has extensive privileges on the local computer, and acts as the computer on the network. Its token includes the NT AUTHORITY\SYSTEM and BUILTIN\Administrators security identifiers; these accounts have access to most system objects.
      The account is not associated with any logged-on user account. This has several implications:
      - The registry key HKEY_CURRENT_USER is associated with the default user, not the current user;
      - The service can open the registry key HKEY_LOCAL_MACHINE\SECURITY;
      - The service presents the computer's credentials to remote servers;
      - If the service opens a command window and runs a batch file, the user could hit CTRL+C to terminate the batch file and gain access to a command window with LocalSystem permissions;
      • Allow service to interact with desktop
        Enable this check box to allow the service to display a user interface and receive user input.
        Typically, services are console applications that are designed to run unattended without a graphical user interface (GUI). However, some services may require occasional interaction with a user.
        However, note that the following registry key contains a value NoInteractiveServices: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Windows. The NoInteractiveServices value defaults to 0 (zero), which means that services are allowed to run interactively. When NoInteractiveServices is set to a nonzero value, no service started thereafter is allowed to run interactively.

  • Load-Order Group

    • Information of an load ordering group of which this service is a member. If the field is disabled, the service does not belong to a group.

    • Name
      The name of the group.

    • Tag in group

      • A unique tag value for this service in the group. Tag can be used for ordering service startup in a load ordering group by specifying a tag order vector in the registry located at: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GroupOrderList
      Tags are only evaluated for Drivers that have Boot or System start types.

  • [View Group]

    • Opens a dialog box that contains list of all members of this group.
